Change Description:
This submission removes ALL dependancies on the old ODT libraries
and code, via #ifdefs. 


To turn the old ODT back on, do the following.//////////////
uncomment 
//IDR_ADINIT		RT_BINDATA	"..\\..\\..\\Licensed\\ODTBIN\\adinit.dat"
in dbdll.rc2

include ADV6ND.lib in db.dsp project settings in ALL configurations.
add 'WITHODT' #define to db.dsp
////////////////////////////////////////////////////////////

Once tested, to completely remove ALL remnents of ODT do the following./////////
remove these directories from the source tree;
......\IntelliCAD\Source\prj\lib\MARCOMP
......\IntelliCAD\Source\Licensed\ODTbin
remove all code between #ifdef WITHODT and the following #else throughout the
source code tree.
////////////////////////////////////////////////////////////

Notes to Developers:
Once you update to this submission, there is no reason to have the
USEDWGDIRECT environment variable set on your system. It will be ignored.

Notes to QA:
As part of my testing, I renamed ......\IntelliCAD\Source\prj\lib\MARCOMP
and ......\IntelliCAD\Source\Licensed\ODTbin directories and did several 
full builds.

Note that a problem has cropped up in exporting ACIS entities to 2004 DWG.
AutoCAD cannot read them. However, AutoCAD can read the DXF files with ACIS
entities in them. I have refered the problem to the ODT guys for help.
