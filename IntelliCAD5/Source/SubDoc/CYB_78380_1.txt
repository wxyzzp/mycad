Description:	"long" data type for TOOLBARID had reached its limits. So it was NOT possible to add new hardcoded toolbars to 
		IntelliCAD. Now we have used new #define  long_double __int64 for TOOLBAR ID. In future one can implement 
		the class long_double and have operator & overloaded and keep the current design of toolbar intact. 
		Other option is to change the current implementation of toolbar for tool id.
Comments:	TaL78380
Problem Analysis:
		'long' data type limit is exhausted. In IntelliCAD code there is one bit of 32 bits of long for every 
		toolbar. 
Problem solution:
		The solution was to get the new data type. As the operation like &, ! are done on the ToolId. We cannot use
		'long double'. Denis suggested a good idea of creating a class and overloding operator &, ! etc so that the
		toolbar design remains intact.
		We can also use __int64 intead of long data type. So the solution is to replace long for __int64.
